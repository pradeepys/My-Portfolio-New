3. Doc X

  • Timeline : 2 weeks

Problem Statement


Is there any way to know what all documents that needs to be carried from one country to another ?

If yes, then how does the user flow looks like, the customer application and specific features to use it ?

How does the user interacts with the application ?

Goal

Creating a mobile application design with all the features required.

Making the User to know what all documents to be carried while travelling to another country.

UI/UX Process

Step 1 : Sketching

Sketching the product how should application looks like on a paper, which makes faster to do any changes. Once all the sketches are finalized, going to Step 2.

Step 2 : Wireframes

Skipped the userflows and directly created low fidelity design wireframes to speed up the process.

Step 3 : Prototyping

Prototyping allow us to create interactive flows that explore how a user may interact with your designs with some cool micro animation. And also we should get the feedback and do necessary changes accordingly before going to step 4.

See some interations here in prototype tab

Step 4 : Visual Design

Visual Design is all about high fidelity design selecting the proper color, icons, typography etc.

Final Designs

All flows at one place,

Conclusion


Successfully designed DocX mobile application, with some specific features.
